What is Docker?

Docker is a platform that uses containerization technology to package applications and their dependencies into lightweight, portable containers. Think of a container as a box that contains everything your application needs to run: code, runtime, system tools, libraries, and settings.

Key Benefits:

  • Consistency: Runs the same way on any machine
  • Efficiency: Uses fewer resources than virtual machines
  • Portability: Move applications easily between environments
  • Scalability: Easy to scale applications up or down

Real-World Analogy

Imagine Docker containers like shipping containers. Just as shipping containers can hold different goods but all fit on the same ship, Docker containers can hold different applications but all run on the same host system.

How Docker Works

Docker uses a client-server architecture with three main components:

Docker Engine

The core runtime that manages containers on your system

Docker Images

Read-only templates used to create containers (like blueprints)

Docker Containers

Running instances of Docker images (like actual buildings from blueprints)

Basic Workflow

  1. Create or download a Docker image
  2. Run the image to create a container
  3. The container runs your application
  4. Stop, start, or remove containers as needed

Getting Started with Docker

Installation

Download Docker Desktop from the official website for your operating system:

  • Windows/Mac: Docker Desktop provides a user-friendly interface
  • Linux: Install Docker Engine directly

Verify Installation

docker --version

Test Your Installation

Run the famous "Hello World" container:

docker run hello-world

This command downloads and runs a simple test container that prints a welcome message.

Running Docker Images

Basic Run Command

docker run [OPTIONS] IMAGE [COMMAND]

Common Examples

Run Ubuntu Container

docker run -it ubuntu bash

-it makes the container interactive with a terminal

Run Web Server (Nginx)

docker run -d -p 8080:80 nginx

-d runs in background, -p 8080:80 maps port 8080 to container's port 80

Run with Volume Mount

docker run -v /host/path:/container/path image-name

Mounts a directory from your host into the container

Useful Run Options

Option Description
-d Run in detached mode (background)
-p Port mapping (host:container)
-v Volume mounting
--name Give container a custom name
--rm Remove container when it stops

Building Docker Images

Dockerfile

A Dockerfile is a text file containing instructions to build a Docker image. It's like a recipe for your container.

Sample Dockerfile

# Use official Node.js runtime as base image FROM node:16 # Set working directory in container WORKDIR /app # Copy package.json and package-lock.json COPY package*.json ./ # Install dependencies RUN npm install # Copy application code COPY . . # Expose port 3000 EXPOSE 3000 # Command to run when container starts CMD ["npm", "start"]

Build the Image

docker build -t my-app:latest .

-t tags the image with a name, . refers to current directory

Common Dockerfile Instructions

FROM: Specifies base image
WORKDIR: Sets working directory
COPY: Copies files from host to container
RUN: Executes commands during build
EXPOSE: Documents which ports the container uses
CMD: Default command when container starts

Docker Hub: Push and Pull Images

What is Docker Hub?

Docker Hub is a cloud-based registry service where you can store and share Docker images. It's like GitHub for Docker images.

Creating an Account

  1. Go to hub.docker.com
  2. Sign up for a free account
  3. Verify your email address

Login from Command Line

docker login

Enter your Docker Hub username and password when prompted.

Tagging Images for Docker Hub

docker tag my-app:latest username/my-app:latest

Replace username with your Docker Hub username.

Push Image to Docker Hub

docker push username/my-app:latest

Pull Image from Docker Hub

docker pull username/my-app:latest

Pro Tip:

Use meaningful tags like v1.0, production, or staging instead of just latest for better version control.

Common Docker Commands

Image Commands

docker images List all images
docker pull image-name Download an image
docker rmi image-name Remove an image
docker build -t name . Build image from Dockerfile

Container Commands

docker ps List running containers
docker ps -a List all containers
docker stop container-id Stop a container
docker start container-id Start a stopped container
docker rm container-id Remove a container
docker exec -it container-id bash Access running container

System Commands

docker system prune Clean up unused resources
docker logs container-id View container logs
docker info Display system information
